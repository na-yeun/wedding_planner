# 💍 Wedding Planner

## 📄 요구사항 정의서

### 공통
1.  회원가입
- 사용자는 메인 화면의 "회원가입 하기" 버튼을 통해 회원가입을 진행할 수 있다.
- 사용자는 회원가입 진행시 ID, 비밀번호, 이름, 본식 날짜를 입력하여 회원가입을 진행할 수 있다.
- 희망할 경우 카카오톡 회원가입이 가능하며, 카카오톡 회원가입의 경우 oAuth 인증을 진행한 후 ID, 비밀번호, 이름, 본식 날짜를 입력하여 회원가입을 완료한다.
- 카카오톡 계정 인증 버튼을 통해 발급받은 oAuth Access 토큰과 Refresh 토큰은 데이터베이스의 oAuth 테이블에 별도로 저장되어 관리되며 해당 테이블에는 ID가 FK로 참조된다.
- 회원가입 진행시 작성한 정보는 데이터 베이스에 저장된다.

2. 로그인
- 메인 화면 접근 시 아이디와 비밀번호를 입력할 수 있는 UI가 제공된다.
- 로그인 화면에서 제공되는 UI에 ID와와 비밀번호를 입력하고 로그인 버튼을 클릭하면 데이터베이스의 member 테이블에 저장된 회원의 정보와 비교하여 로그인을 진행한다.
- 입력한 정보와 member 테이블에 저장된 사원의 정보가 일치할 경우, 사원의 PK(ID)를 참조하고 있는 oAuth 관련 테이블에서 해당 회원의 oAuth 인증 토큰이 함께 조회되며 Access 토큰이 만료되었을 경우 Refresh 토큰을 통해 재인증이 진행된다.
- Refresh 토큰을 통해 재인증이 진행될 때, 어떠한 이유로 Refresh  토큰이 만료되거나 더 이상 사용할 수 없을 경우에는 다시 카카오 계정 인증 단계로 넘어간다. 다시 카카오 계정 인증을 진행하여 새로운 Access토큰과 Refresh토큰이 발급되면 oAuth 관리 테이블에 업데이트 한다.
- 최종 로그인 성공 시 정보가 세션에 저장되며 회원의 메인 화면(일정)으로 이동한다.
- 최종 로그인 실패시 로그인 실패 안내 메세지를 alert를 통해 출력하고, 로그인 창으로 다시 이동한다.
- ID에 해당하는 비밀번호가 1일 3회 이상 오류 시 본인 확인 페이지로 이동한다.
- 본인 확인 페이지에서는 문자 인증을 진행한다. 문자 인증은 이름, 사원번호, 핸드폰 번호를 입력해야하며 해당 입력 정보가 데이터 베이스에 저장된 데이터와 일치할 경우 발송된다.

3. 로그아웃
- 메인페이지의 "로그아웃" 버튼을 통해 로그아웃을 진행할 수 있다.
- 로그아웃에 성공하면 로그인 정보가 세션에서 삭제되고 메인 페이지(로그인 창)로 이동한다.

4. 비밀번호 찾기
- 비밀번호 찾기 버튼 클릭시 비밀번호 찾기 페이지로 이동한다.
- 비밀번호 찾기 페이지에서는 ID, 핸드폰번호를 입력한다.
- 입력한 정보가 일치할 경우 핸드폰번호로 새로 발급된 비밀번호가 발송된다.


### 마이페이지
1. 개인정보 수정
- "개인정보 수정" 메뉴 클릭시 비밀번호 확인 페이지로 이동한다. 비밀번호 일치시에만 개인정보를 수정할 수 있다. 비밀번호를 바르게 입력한 경우 개인정보 수정 페이지로 이동한다.
- "나의 정보"에 저장된 비밀번호, 핸드폰번호, 본식 날짜를 수정할 수 있다. 신랑 혹은 신부가 짝꿍으로 연결되어있을 경우 본식 날짜를 수정하면 짝꿍의 본식 날짜도 함께 수정된다. 그 외의 이름, ID는 수정할 수 없다.

2. 짝꿍 관리
- 짝꿍 버튼을 통해 신랑 혹은 신부 계정과 짝꿍을 맺거나 짝꿍을 삭제할 수 있다. 짝꿍을 맺을 때는 해당 버튼을 누르고 짝꿍의 ID 혹은 핸드폰 번호를 입력하면 상대에게 짝꿍 요청이 전송된다. 
- 이미 상대방이 짝꿍 요청을 보냈을 경우에는 짝꿍 버튼을 통해 요청을 확인할 수 있으며 승낙 / 거절을 할 수 있다.
- "짝꿍 승낙시 짝꿍 맺기 전 데이터가 존재할 경우, 모두 초기화 / 신랑 데이터만 유지 / 신부 데이터만 유지 / 신랑 신부 데이터 모두 유지 중 선택할 수 있다. 단, 이 때 한 번 선택이 완료되면 이전의 데이터를 복구할 수는 없다."


### 일정관리
1. 일정 조회
- 사용자(신랑, 신부)가 로그인 하면 가장 먼저 일정 화면이 조회된다.
- 일정 상단에는 결혼식 당일 D-day가 조회되며 아래에는 달력이 조회된다.
- 캘린더에서는 사용자가 원하는 일정을 클릭하여 해당 세부 정보를 조회할 수 있어야 한다.
- 사용자는 월간, 주간, 일간 보기 옵션을 통해 일정을 조회할 수 있어야 하며, 이를 통해 일정 관리와 시각화에 유연성을 제공할 수 있다.
- 일정을 클릭하면 상세 보기 화면이 표시되어야 하며, 캘린더 일정의 제목, 설명, 날짜 및 시간, 위치와 같은 상세 정보를 조회 할 수 있다.
- 시스템은 일정이 설정된 날짜와 시간에 Toast UI 알림을 통해 사용자에게 알림을 제공해야 한다. 알림은 로그인시, 혹은 적시에 전달되어야 하며, 사용자의 빠른 일정 확인을 위해 알림창에 일정 제목이나 세부 정보를 포함하여 보여줘야 한다.

2. 일정 작성 및 관리
- 사용자는 일정 추가 버튼을 클릭하면 새로운 일정을 만들 수 있으며, 입력해야 할 세부 사항은 다음과 같다: 제목, 설명, 시작 날짜 및 시간, 종료 날짜 및 시간, 위치 / 그 중 필수 입력 항목은 제목, 시작 날짜 및 시간, 종료 날짜 및 시간이다
- 일정을 등록하면 짝꿍(신랑 혹은 신부) 모두 해당 일정을 조회할 수 있다.
- 캘린더 인터페이스는 드래그 앤 드롭 기능을 지원하여 사용자가 이벤트를 새로운 시간대로 이동시켜 일정을 변경할 수 있다. 삭제는 지정된 삭제 영역으로 드래그하거나 삭제 옵션을 사용하여 수행할 수 있다.
- 일정의 수정은 등록자와 상관없이 신랑, 신부 모두 수정 가능하다.
- 수정 가능한 항목은 제목, 설명, 날짜 및 시간, 위치, 참가자 등이다.
- 사용자는 캘린더에서 일정을 선택하고 삭제 옵션을 사용하여 일정을 삭제할 수 있어야 하며, 삭제 시 확인 메시지가 표시되어야 한다.
- 사용자가 일정 등록 또는 수정 시 기존 일정과 시간이 중복될 경우, 시스템은 alert 알림창을 통해 일정 중복 경고가 표시돼야 한다. 사용자는 충돌을 해결한 후에야 일정을 저장할 수 있다.


### TODO
1. TODO 조회
- 사용자는 완료처리가 되지 않은 순서로 To do list를 조회할 수 있다.
- To do list는 짝꿍이 작성한 리스트까지 포함되어 표시되며, 중요도 순서로 조회된다.
- 완료처리가 된 리스트는 가장 하단에 표시된다.

2. TODO 관리
- 내용과 중요도를 설정하여 추가 및 수정할 수 있다.
- 이미 작성된 리스트를 수정할 때는 수정 버튼을 누르면 수정창이 열린다.
- 리스트 완료처리를 하면 체크박스 체크, 취소선이 생기며 가장 아래로 내려간다.
- 완료처리된 리스트를 다시 클릭하면 완료처리가 취소된다.


### 지출관리
1. 지출관리 조회
- 사용자는 서로가 작성한 지출을 조회할 수 있다.
- 지출 조회시에 전체 합계를 함께 조회할 수 있으며 지출 선택시 지출에 대한 상세내역을 조회할 수 있다.

2. 지출 작성 및 수정(관리)
- 사용자는 내용, 카테고+D45리, 예산, 실 결제 금액, 계약금, 잔금, 상태, 결제 수단을 작성하여 지출내역을 작성할 수 있다.
- 지출의 카테고리는 기본으로 웨딩홀, 스드메, 혼주, 신혼여행을 제공하며, 기본 카테고리를 포함하여 모든 카테고리는 추가와 수정, 삭제가 가능하다.
- 결제 수단의 카테고리는 기본으로 현금결제, 카드결제를 제공하며, 기본 카테고리를 포함하여 모든 카테고리는 추가와 수정, 삭제가 가능하다.
- 상태의 카테고리는 기본으로 진행 전, 계약금 완료, 잔금 완료를 제공하며, 기본 카테고리를 포함하여 모든 카테고리는 추가와 수정, 삭제가 가능하다.
